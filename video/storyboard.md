## Recording Order

Record these groups separately:

## Editing Notes

- Keep narration as the timing source.
- Trim dead time aggressively.
- Do not leave any static page on screen for more than `6-10s` without a visual change.
- Prefer short purposeful scrolls over long reads.
- Use captions sparingly and keep them away from important UI regions.
- In Scene 6, do not jump from wiki home straight to GitHub or the repo. The viewer must see a readable `catalog -> detail` progression.

## Animation Priority

If time is tight, only keep these motion inserts:

1. `run -> review -> verification -> promotion`
2. PI agent gated handoff flow
3. directory path highlight in repo architecture section
