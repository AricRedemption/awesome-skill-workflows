## Script

### 1. Hook

This is not a prompt collection.  
Runwiser is a framework for turning repeated agent work into verified, reusable workflow assets.

### 2. What Runwiser Is

Instead of saving only the final output, this project preserves the workflow itself.  
Skills, evidence, review gates, verification, and reusable knowledge stay connected.

### 3. Why Proof Comes First

Most agent systems reuse a result as soon as it looks good.  
Runwiser separates those states.  
A run is evidence, not truth.  
Assets are promoted only after proof, review, and validation line up.

### 4. PI Agent Orchestration

This project is also about how multiple agents work together inside one workflow.  
Not as loose parallelism, but as role-based orchestration.  
Research, composition, review, verification, and writeback move through gated handoffs, so the system preserves not just what was produced, but how it was checked.

### 5. Wiki On The Landing Page

On the landing page, the wiki appears as the public surface of the system.  
It signals that the knowledge layer is meant to be searched, browsed, and reused.

### 6. Live Wiki Service

Then the live wiki service makes that promise concrete.  
You can open the catalog, move into a specific page, and browse promoted knowledge as a real working surface.

### 7. Repo Architecture

The repository is structured around that lifecycle.  
Runs store concrete evidence.  
Workflows and schemas define reusable contracts.  
The knowledge base stores durable lessons.  
Scenarios validate the architecture in real operating contexts.

### 8. Proof In Practice

So this is not just an idea with a polished landing page.  
You can see the proof directly in the repo.  
Runs, workflows, knowledge assets, validation scripts, and scenario wrappers live in separate, explicit layers.

### 9. Close

That is the core idea behind Runwiser.  
Make agent work reusable, measurable, and improvable by preserving evidence, enforcing gates, and promoting only what has truly been verified.
