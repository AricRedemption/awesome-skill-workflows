## Script

### 1. Hook

This is Runwiser.  
It turns repeated agent work into reusable workflow assets, with proof attached.

### 2. Proof First

The landing page leads with the operating model.  
Runs create evidence, gates check quality, and only verified work moves forward.

### 3. PI Agent Orchestration

This is not loose multi-agent parallelism.  
Five specialized roles move work through research, compose, review, verify, and writeback.

### 4. Live Wiki Service

The live wiki service makes the knowledge layer concrete.  
You can open the catalog, move into a detail page, and browse promoted workflow knowledge directly.

### 5. Architecture Overview

The system is organized into five layers around one promotion loop.  
Evidence starts at layer zero, reusable skills live at layer one, and optimization stays in layer two.

### 6. Architecture Boundary

Layer three becomes durable knowledge, while layer four keeps scenario validation local.  
That boundary matters, because scenarios can validate the architecture, but they should not define it.

### 7. Proof In The Repo

You can see that separation directly in the repo.  
Runs, workflows, scripts, knowledge assets, and scenario wrappers live in different places for different jobs.

### 8. Close

That is the point of Runwiser.  
Make agent work measurable, improvable, and reusable by preserving evidence before promotion.
