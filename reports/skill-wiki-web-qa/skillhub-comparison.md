# SkillHub Comparison Notes

## Compared Surfaces

- Target: `https://skillhub.cn/` homepage screenshot
- Local: `skill-wiki-web/` homepage and catalog screenshots

## Target Product Grammar

The current SkillHub homepage behaves like a discovery product, not a document
site:

- centered hero instead of left-heavy explanation layout
- one dominant value statement
- compact primary actions
- lightweight install/search card under the hero
- immediate transition into searchable or recommended catalog content
- cards prioritize title, short summary, and lightweight metadata

## What Is Now Aligned

- centered hero with dominant headline
- compact CTA grouping
- lighter home information density
- product-style catalog cards instead of long document tiles
- collapsed tag wall on catalog page
- reduced summary and tag density on cards
- standalone discovery flow rather than embedding inside the old landing page

## What Still Differs

- typography is closer, but not a close clone of SkillHub's font personality
- target uses richer product chrome such as tab states, mascot illustration, and
  more opinionated marketplace cues
- local homepage still contains repository-truth framing that the target product
  does not need
- local detail pages are intentionally evidence-heavy because the Skill Wiki data
  model carries workflow proof and provenance

## Current Judgment

The local site is now materially closer to SkillHub's discovery layout and
interaction rhythm than the earlier document-like version. It is suitable for
functional QA and repository-backed browsing, but it is still a product-inspired
clone rather than a pixel-accurate visual duplicate.
