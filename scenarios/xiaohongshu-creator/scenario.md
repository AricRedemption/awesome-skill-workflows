# Xiaohongshu Creator Scenario

## Scenario

- scenario name: `xiaohongshu-creator v0.1`
- platform: `xiaohongshu`
- account direction: `AI tools / productivity`
- target users: `knowledge workers, independent creators, small startup teams`
- task 001: `How ordinary users can save one hour per day with AI agents`
- task 002: `AI meeting-notes productivity workflow`

## Purpose

This scenario validates whether the core skill-workflow architecture can support a real account-bound content workflow without letting that scenario become the architecture itself.

## Required Capabilities

1. discover and normalize candidate skills,
2. map scenario capabilities,
3. compose a workflow,
4. generate a content package,
5. score quality and compliance,
6. collect human review,
7. check account state,
8. hand off to draft or publish mode only after approval,
9. capture proof,
10. write reusable evidence back to the workflow knowledge base.

## Boundaries

- Xiaohongshu assumptions stay in this scenario and related run evidence.
- Generic skills and workflows must remain reusable for other content platforms.
- Publishing and draft actions require explicit review and account-state gates.
