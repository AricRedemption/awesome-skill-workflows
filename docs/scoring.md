# Scoring

This document defines the initial scoring direction for workflows and skills.

## Scoring Goals

- judge whether a workflow is reusable
- judge whether a skill is stable enough to promote
- judge whether a scenario run produced knowledge worth keeping
- identify failure patterns that should become explicit recipes or guardrails

## Initial Score Dimensions

- usefulness
- repeatability
- human review burden
- failure severity
- reuse potential

## MVP Rule

A workflow is only considered verified when the run outcome and review feedback are written into the workflow knowledge base.
