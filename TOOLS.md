# Tools Notes

This file is reserved for durable tool, command, environment, path, and platform
usage notes that should affect future runs.

Use `TOOLS.md` for:

- command flags or invocation patterns that must be reused,
- local environment constraints,
- validator or script caveats,
- external tool behavior that caused a repeatable failure,
- paths or runtime setup notes that are not user preferences.

Do not use this file for:

- run-specific evidence,
- scenario-specific platform rules,
- sensitive values,
- cookies, tokens, browser profile paths, or credentials.

Tool notes should be added through an approved draft under `evolution-drafts/`
when they are long-lived. Run-specific tool evidence belongs in `runs/` or
`evolution/`.
